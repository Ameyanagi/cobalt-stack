1//! Authentication service layer.
2//!
3//! This module provides business logic for user authentication, including
4//! JWT token generation/verification, password hashing, and secure token rotation.
5//! It follows Domain-Driven Design principles with clear separation between
6//! the service layer (business logic) and handler layer (HTTP concerns).
7//!
8//! # Architecture
9//!
10//! The authentication service is organized into submodules:
11//!
12//! - **error**: Domain-specific error types and HTTP mapping
13//! - **jwt**: JSON Web Token creation and verification
14//! - **password**: Secure password hashing and verification with Argon2
15//! - **token_rotation**: Refresh token rotation and revocation
16//!
17//! # Security Features
18//!
19//! - **Argon2 password hashing**: Industry-standard password protection
20//! - **JWT tokens**: Stateless authentication with access/refresh tokens
21//! - **Token rotation**: Automatic refresh token rotation prevents theft
22//! - **Token revocation**: Individual token revocation capability
23//! - **Hash storage**: Refresh tokens stored as SHA-256 hashes
24//!
25//! # Token Flow
26//!
27//! ```text
28//! 1. Register/Login → Generate access token (30m) + refresh token (7d)
29//! 2. Access protected resources with access token
30//! 3. Access token expires → Use refresh token to get new pair
31//! 4. Refresh token used → Old token revoked, new token issued (rotation)
32//! 5. Logout → Refresh token revoked
33//! ```
34//!
35//! # Examples
36//!
37//! ```no_run
38//! use cobalt_stack::services::auth::{
39//!     hash_password, verify_password, create_access_token,
40//!     create_refresh_token, JwtConfig
41//! };
42//! use uuid::Uuid;
43//!
44//! # async fn example() -> Result<(), Box<dyn std::error::Error>> {
45//! // Password hashing
46//! let hash = hash_password("user_password")?;
47//! assert!(verify_password("user_password", &hash)?);
48//!
49//! // JWT token generation
50//! let config = JwtConfig::from_env();
51//! let user_id = Uuid::new_v4();
52//! let access_token = create_access_token(user_id, "alice".to_string(), &config)?;
53//! let (refresh_token, jti) = create_refresh_token(user_id, &config)?;
54//! # Ok(())
55//! # }
56//! ```
57//!
58//! # Error Handling
59//!
60//! All service functions return [`Result<T>`] using domain-specific [`AuthError`] types.
61//! Errors are automatically mapped to appropriate HTTP status codes via `IntoResponse`.
62
