Database entity models and domain types.

This module contains SeaORM entity definitions that map to database tables in the PostgreSQL database. The entities are generated by sea-orm-codegen and represent the core domain models for authentication and user management.

§Architecture

The models follow Domain-Driven Design principles:

  • Each entity represents a database table
  • Relations define foreign key relationships
  • ActiveModelBehavior provides lifecycle hooks
  • Entities are immutable; use ActiveModel for mutations

§Database Mapping

  • users: User accounts with authentication credentials
  • refresh_tokens: JWT refresh tokens for token rotation
  • email_verifications: Email verification tokens and status
  • o_auth_accounts: OAuth provider account linkages

§Entity Relations

Users (1) ──< (N) RefreshTokens
      (1) ──< (N) EmailVerifications
      (1) ──< (N) OAuthAccounts

§Examples

use cobalt_stack::models::prelude::*;
use sea_orm::*;

// Query a user with related refresh tokens
let user = Users::find_by_id(user_id)
    .find_with_related(RefreshTokens)
    .all(db)
    .await?;

§Code Generation

These entities are generated from the database schema using:

sea-orm-cli generate entity \
  -u postgres://user:pass@localhost/db \
  -o src/models
